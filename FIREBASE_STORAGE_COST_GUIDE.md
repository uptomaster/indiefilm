# Firebase Storage 비용 가이드

## 💰 무료 할당량 (Blaze 플랜)

Firebase Storage는 **Blaze 플랜으로 업그레이드해도 무료 할당량 내에서는 요금이 부과되지 않습니다.**

### 무료 할당량 상세

| 항목 | 무료 할당량 | 월간 무료 |
|------|------------|----------|
| **저장** | 5GB | 5GB |
| **다운로드** | 1GB/일 | 약 30GB/월 |
| **업로드** | 20GB/일 | 약 600GB/월 |
| **작업** | 50,000회/일 | 약 1,500,000회/월 |

### 실제 사용량 추정

#### 이미지 크기 가정
- 프로필 사진: 평균 200KB
- 갤러리 이미지: 평균 500KB
- 영화 썸네일: 평균 300KB

#### 시나리오별 사용량

**시나리오 1: 소규모 서비스 (배우 100명, 영화 50개)**
- 프로필 사진: 100명 × 200KB = 20MB
- 갤러리 이미지: 100명 × 5장 × 500KB = 250MB
- 영화 썸네일: 50개 × 300KB = 15MB
- **총 저장량: 약 285MB** ✅ (5GB의 5.7%)

**시나리오 2: 중규모 서비스 (배우 500명, 영화 200개)**
- 프로필 사진: 500명 × 200KB = 100MB
- 갤러리 이미지: 500명 × 5장 × 500KB = 1.25GB
- 영화 썸네일: 200개 × 300KB = 60MB
- **총 저장량: 약 1.41GB** ✅ (5GB의 28%)

**시나리오 3: 대규모 서비스 (배우 2,000명, 영화 1,000개)**
- 프로필 사진: 2,000명 × 200KB = 400MB
- 갤러리 이미지: 2,000명 × 5장 × 500KB = 5GB
- 영화 썸네일: 1,000개 × 300KB = 300MB
- **총 저장량: 약 5.7GB** ⚠️ (5GB 초과)

### 다운로드 사용량 추정

**일일 조회 가정:**
- 배우 프로필 조회: 1,000회/일 × 200KB = 200MB
- 갤러리 이미지 조회: 500회/일 × 500KB = 250MB
- 영화 썸네일 조회: 2,000회/일 × 300KB = 600MB
- **총 다운로드: 약 1.05GB/일** ⚠️ (1GB 초과 가능)

## 💡 비용 최적화 전략

### 1. 이미지 최적화
- 이미지 압축 (WebP 형식 사용)
- 적절한 해상도 (프로필: 800x800px, 썸네일: 1280x720px)
- 지연 로딩 (Lazy Loading) 구현

### 2. CDN 활용
- Firebase Storage는 자동으로 CDN을 사용
- 자주 조회되는 이미지는 캐시됨

### 3. 하이브리드 방식
- 자주 조회되는 이미지: Firebase Storage
- 덜 조회되는 이미지: 외부 호스팅 (Imgur 등)

### 4. 사용량 모니터링
- Firebase 콘솔 → 사용량 및 청구
- 일일 사용량 확인
- 알림 설정 (무료 할당량의 80% 도달 시)

## 📊 실제 비용 계산

### 무료 할당량 초과 시 (GB 단위)

**저장 비용:**
- 5GB 초과분: $0.026/GB/월
- 예: 10GB 저장 시 → (10-5) × $0.026 = $0.13/월

**다운로드 비용:**
- 1GB/일 초과분: $0.12/GB
- 예: 2GB/일 다운로드 시 → (2-1) × $0.12 = $0.12/일 = $3.6/월

**업로드 비용:**
- 20GB/일 초과분: $0.05/GB
- 예: 25GB/일 업로드 시 → (25-20) × $0.05 = $0.25/일 = $7.5/월

## ✅ 결론

### 포트폴리오/소규모 서비스
- **무료 할당량으로 충분히 사용 가능**
- Blaze 플랜 업그레이드 후 무료로 사용 가능
- 결제 정보만 등록하면 됨 (실제 요금은 무료 할당량 내에서 발생하지 않음)

### 중규모 서비스
- 저장량은 무료 할당량 내에서 가능
- 다운로드가 많으면 약간의 비용 발생 가능 ($1-5/월 정도)

### 대규모 서비스
- 이미지 최적화 필수
- 하이브리드 방식 고려
- 사용량 모니터링 및 알림 설정
