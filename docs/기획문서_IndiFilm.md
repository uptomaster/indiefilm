# IndiFilm 기획문서

> 인디 영화 제작자와 배우 지망생을 위한 네트워킹 플랫폼

**문서 목적**: IndiFilm 프로젝트의 기능, 아키텍처, 확장가능성 및 로드맵을 이해하고 타인에게 설명하기 위한 기획 문서

---

## 1. 프로젝트 개요

### 1.1 비전 및 미션

IndiFilm은 **대학생 동아리, 인디 영화 제작자, 배우 지망생**이 서로를 찾고 협업할 수 있도록 돕는 **통합 네트워킹 플랫폼**입니다. 별도의 커뮤니티나 SNS 없이도 영화 제작, 캐스팅, 포트폴리오 관리, 커뮤니케이션을 한 곳에서 진행할 수 있는 **올인원 플랫폼**을 목표로 합니다.

### 1.2 핵심 가치 제안

| 대상 | 제공 가치 |
|------|-----------|
| **배우 지망생** | 프로필 관리, 캐스팅 정보 접근, 제작자와의 직접 소통, 포트폴리오(데모릴, 갤러리) 관리 |
| **인디 제작자** | 배우 검색·캐스팅, 영화 포트폴리오 관리, 팀 프로필, 커뮤니티를 통한 협업 모집 |
| **일반 시청자** | 인디 영화 관람, 평점·리뷰 참여, 제작자·배우 탐색 |

### 1.3 현재 성숙도

- **웹**: 핵심 기능 구현 완료, 운영 가능 수준
- **모바일**: Expo 기반 프로젝트 구성만 완료, 기능 미구현
- **인프라**: Firebase 기반, Vercel 배포 구성 완료

## 2. 주요 기능

### 2.1 현재 구현된 기능

#### 인증 및 역할
- 이메일/비밀번호 로그인
- Google OAuth 로그인
- 회원가입 후 역할 선택 (배우 / 제작자 / 시청자)

#### 배우 관련
- 프로필 생성·수정·조회
- 검색 및 필터링 (지역, 나이대 등)
- 육각형 특성 차트 (연기력, 외모, 카리스마, 감성, 유머)
- 갤러리 및 데모릴 업로드
- 출연 작품(크레딧) 관리

#### 영화 및 제작자
- 영화 업로드, 수정, 목록/상세 조회
- 영화 상태 (제작중 / 제작예정 / 제작완료)
- 장르별 필터링
- 제작자/팀 프로필 관리
- 필모그래피 관리

#### 커뮤니티
- 게시글 유형: 캐스팅 공고, 배우 구직, 일반
- 댓글 기능
- 조회수 기록

#### 요청 및 채팅
- 캐스팅 제안 / 출연 희망 요청
- 요청 상태 (대기 / 수락 / 거절)
- 요청 단위 1:1 실시간 채팅

#### 기타
- 통합 검색 (영화, 배우, 제작자, 게시글)
- 영화 평점 및 리뷰
- 인생영화 표시
- 읽지 않은 요청 카운트 (알림 UI)
- Toast 알림, 에러 바운더리, 스켈레톤 로더

### 2.2 페이지 구조

| 경로 | 설명 |
|------|------|
| `/` | 홈 (히어로, 기능 소개) |
| `/login`, `/signup` | 로그인/회원가입 |
| `/role-select` | 역할 선택 |
| `/actors` | 배우 목록 |
| `/actors/[id]` | 배우 상세 |
| `/actors/me`, `edit`, `view` | 내 배우 프로필 |
| `/movies` | 영화 목록 |
| `/movies/[id]` | 영화 상세 |
| `/movies/new`, `[id]/edit` | 영화 생성/수정 |
| `/movies/ratings` | 내 영화 평점 |
| `/filmmakers` | 제작자 목록 |
| `/filmmakers/[id]`, `me`, `edit`, `view` | 제작자 프로필 |
| `/posts` | 커뮤니티 게시판 |
| `/posts/new`, `[id]` | 게시글 작성/상세 |
| `/requests` | 요청 목록 |
| `/requests/[id]` | 요청 상세 및 채팅 |
| `/search` | 통합 검색 |
| `/credits` | 크레딧(출연 작품) 브라우징 |

## 3. 기술 아키텍처

### 3.1 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| **웹** | Next.js 16 (App Router), React 19, TypeScript | 서버/클라이언트 |
| **스타일** | Tailwind CSS 4, shadcn/ui, Radix UI | |
| **폼** | React Hook Form, Zod, @hookform/resolvers | |
| **상태** | Zustand, React Context, 커스텀 훅 | |
| **DB** | Firebase Firestore | NoSQL |
| **인증** | Firebase Authentication | 이메일, Google |
| **스토리지** | Firebase Storage | 이미지, 영상 |
| **호스팅** | Vercel | 웹 |
| **모바일** | React Native, Expo 54 | 현재 스켈레톤 수준 |

### 3.2 데이터 모델 (Firestore)

| 컬렉션 | 용도 | 주요 필드 |
|--------|------|-----------|
| `users` | 사용자 프로필 | uid, email, displayName, role |
| `actors` | 배우 프로필 | userId, stageName, ageRange, skills, traits |
| `movies` | 영화 | title, genre, status, filmmakerId, credits |
| `filmmakers` | 제작자 프로필 | type, name, bio |
| `posts` | 커뮤니티 게시글 | authorId, type, title, content |
| `posts/{id}/comments` | 댓글 | userId, content |
| `movieRatings` | 영화 평점 | userId, movieId, rating, review |
| `requests` | 캐스팅/출연 요청 | fromUserId, toUserId, status |
| `requests/{id}/messages` | 채팅 메시지 | userId, message |

### 3.3 데이터 접근 패턴

- **API 서버 없음**: 클라이언트에서 Firebase SDK로 직접 접근
- **실시간**: 요청 카운트, 채팅 메시지 등 Firestore 리스너 사용
- **필터링/정렬**: 상당 부분 클라이언트에서 처리 (Firestore 쿼리 제약 고려)

## 4. 확장 가능성

### 4.1 기능 확장

| 확장 영역 | 가능한 기능 | 난이도 |
|-----------|-------------|--------|
| **검색** | 고급 필터, 자동완성, Algolia/Elasticsearch 연동 | 중 |
| **알림** | FCM 푸시, 알림 센터, 타입별 필터 | 중 |
| **소셜** | 팔로우/좋아요, 북마크, 활동 피드 | 중 |
| **분석** | 프로필 조회수, 요청 통계, 트렌드 | 중 |
| **AI** | 추천, 유사 콘텐츠, 태그 자동화 | 상 |
| **관리자** | 콘텐츠 모더레이션, 사용자 관리, 대시보드 | 상 |

### 4.2 규모 확장 (Scalability)

| 영역 | 현재 | 확장 방안 |
|------|------|-----------|
| **검색** | 클라이언트 필터링, 100건 수준 | Algolia/Elasticsearch, Firestore 텍스트 검색 |
| **목록** | 클라이언트 정렬, 전체 로드 | 커서 기반 페이지네이션, 가상화 |
| **이미지** | 원본 업로드, 5MB 제한 | 리사이징, WebP, CDN |
| **채팅** | Firestore 실시간 리스너 | 현재 구조 유지 또는 전문 채팅 서비스 연동 |
| **모바일** | 미구현 | 웹 기능 동등 구현, 공통 로직 공유 |

### 4.3 기술적 확장 포인트

- **lib 모듈 구조**: 영역별 분리(actors, movies, posts 등)로 새 도메인 추가 용이
- **공통 훅**: `useAuth`, `useNotifications` 등 재사용 가능
- **역할 기반 접근**: `actor`, `filmmaker`, `viewer` 역할에 따른 UI/권한 분기
- **서버 API 도입 시**: Next.js API 라우트 또는 BFF 레이어로 점진적 전환 가능

## 5. 로드맵 및 우선순위

### 5.1 우선순위 1: 핵심 UX (단기)

- 통합 검색바 및 고급 필터
- 실시간 알림 센터
- 에러 처리·로딩 상태 개선
- Toast, 에러 바운더리 등 사용자 피드백 강화

### 5.2 우선순위 2: 기능 강화 (중기)

- 정렬·필터링 개선 (URL 쿼리, 저장 필터)
- 이미지 최적화 (리사이징, WebP, lazy loading)
- React.memo, 가상화 등 성능 최적화
- 데이터 검증 강화

### 5.3 우선순위 3: 사용자 경험 (중장기)

- 빈 상태 UI 통일
- 접근성 개선 (키보드, 스크린 리더)
- 반응형·모바일 최적화
- 온보딩·가이드

### 5.4 우선순위 4: 고급 기능 (장기)

- 대시보드 및 통계
- 팔로우, 좋아요, 북마크
- AI 기반 추천
- 관리자 도구

## 8. 요약

IndiFilm은 인디 영화 생태계를 위한 **올인원 네트워킹 플랫폼**으로, 배우·제작자·시청자의 연결과 협업을 지원합니다. Next.js와 Firebase 기반의 웹 앱은 핵심 기능을 갖추었으며, 모듈형 구조와 역할 기반 설계를 통해 검색·알림·소셜·AI 등으로의 확장이 용이합니다. 단기적으로는 검색·알림·UX 개선에, 장기적으로는 모바일 앱과 고급 기능 확장에 집중하는 것이 효과적입니다.
