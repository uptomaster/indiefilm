# Firebase Storage 해결 방법

## 🔍 문제 상황
Firebase Storage를 사용하려면 Blaze 플랜(종량제)으로 업그레이드해야 합니다.

## ✅ 해결 방법

### 방법 1: Blaze 플랜 업그레이드 (권장)

**장점:**
- 썸네일/프로필 사진을 바로 사용 가능
- 완전한 기능 구현 가능
- 무료 할당량이 충분함 (초기 사용자 기준)

**단계:**

1. **Firebase 콘솔에서 업그레이드**
   - Firebase 콘솔 → 프로젝트 설정 → 사용량 및 결제
   - "Blaze 플랜으로 업그레이드" 클릭
   - 결제 정보 입력 (신용카드 등록 필요)

2. **Budget Alert 설정 (중요!)**
   - Firebase 콘솔 → 사용량 및 결제 → 예산 및 알림
   - 예산 만들기
   - 예산 금액: $5 또는 $10 (원하는 금액)
   - 알림 임계값: 50%, 90%, 100%
   - 이메일 알림 설정

3. **Storage 활성화**
   - Firebase 콘솔 → Storage
   - "시작하기" 클릭
   - 위치 선택 (asia-northeast3 - 서울 권장)
   - Security Rules 설정 (`storage.rules` 파일 내용 복사)

**무료 할당량 (매일):**
- 다운로드: 1GB
- 업로드: 20,000건
- 저장: 5GB (총)

**예상 비용:**
- 초기 1,000~5,000 사용자: **월 $0~5** (무료 할당량 내)
- 사용자 증가 시: 사용량에 따라 과금

---

### 방법 2: Storage 없이 먼저 진행 (임시)

**장점:**
- 지금 바로 개발 시작 가능
- 비용 걱정 없음

**단점:**
- 썸네일/프로필 사진 기능 없음
- 나중에 Storage 추가 시 코드 수정 필요

**적용 방법:**
- 영화 업로드 시 썸네일 필드 제외
- 배우 프로필에서 사진 필드 제외
- 텍스트 기반 기능만 먼저 구현
- 나중에 Storage 추가 후 이미지 기능 추가

---

## 💡 추천

**초기 MVP 단계라면:** 방법 2 (Storage 없이 진행)
- 빠르게 핵심 기능 구현
- 사용자 테스트 후 Storage 추가

**정식 출시 목표라면:** 방법 1 (Blaze 업그레이드)
- 완전한 기능 구현
- Budget Alert로 비용 제어
