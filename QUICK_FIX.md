# 빠른 해결 가이드

## 🔴 Firebase: Error (auth/configuration-not-found)

### 즉시 해결 방법

#### 1. 개발 서버 완전히 재시작
```powershell
# 터미널에서 Ctrl+C로 서버 중지
# 그 다음 다시 시작
cd web
npm run dev
```

#### 2. .env.local 파일 위치 확인
파일이 **반드시** `web/.env.local` 위치에 있어야 합니다:
```
IndiFilm/
  web/
    .env.local  ← 여기!
    package.json
    ...
```

#### 3. 파일 내용 확인
브라우저 콘솔(F12)에서 다음 메시지 확인:
- "❌ Firebase 환경변수가 누락되었습니다" → 환경변수 문제
- 다른 에러 메시지 → 해당 에러 해결

#### 4. 환경변수 직접 확인
터미널에서:
```powershell
cd web
# Windows PowerShell
Get-Content .env.local
```

모든 값이 제대로 있는지 확인!

## 🔴 역할 선택 시 변화 없음

### 해결됨! ✅
이제 역할을 클릭하면:
- ✅ 선택된 역할이 파란색으로 강조됨
- ✅ 배경색이 변경됨
- ✅ 약간 확대됨 (scale-105)
- ✅ 로딩 스피너 표시됨

### 테스트 방법
1. Google 로그인 또는 회원가입
2. 역할 선택 페이지로 이동
3. 역할 카드 클릭
4. 즉시 시각적 변화 확인 (파란색 강조)
5. 로딩 스피너 확인
6. 자동으로 홈으로 이동

## 🆘 여전히 안 되면

### 체크리스트
- [ ] 개발 서버 재시작 완료
- [ ] `web/.env.local` 파일 존재 확인
- [ ] 파일 내용이 올바른지 확인
- [ ] 브라우저 콘솔(F12) 에러 메시지 확인
- [ ] Firebase 콘솔에서 프로젝트 설정 확인

### 브라우저 콘솔 확인
F12 → Console 탭:
- 빨간색 에러 메시지 확인
- "Firebase 환경변수 누락" 메시지 확인
- 다른 에러 메시지 확인

### Firebase 콘솔 확인
1. https://console.firebase.google.com
2. 프로젝트 선택: `indiefilm-hub`
3. 프로젝트 설정 (톱니바퀴 아이콘)
4. 내 앱 → 웹 앱 설정 확인
5. SDK 설정 코드와 `.env.local` 값 비교

## 💡 팁

### 환경변수가 제대로 로드되지 않을 때
Next.js는 환경변수를 빌드 타임에 읽습니다. 따라서:
1. `.env.local` 파일 수정
2. **반드시** 개발 서버 재시작
3. 브라우저 캐시 클리어 (Ctrl+Shift+R)

### 역할 선택이 안 될 때
1. 브라우저 콘솔 확인 (F12)
2. "역할 선택 시작: [역할]" 메시지 확인
3. "프로필 생성 완료" 메시지 확인
4. Firestore 콘솔에서 `users` 컬렉션 확인
