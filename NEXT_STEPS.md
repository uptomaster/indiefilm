## 🔧 Firebase 콘솔에서 해야 할 일

### 1. Firestore Security Rules 설정 (필수)

1. Firebase 콘솔 → Firestore Database → 규칙
2. `firestore.rules` 파일 내용 복사 → 붙여넣기 → 게시

### 2. Firestore 인덱스 생성 (필수)

영화 목록 페이지의 장르별 필터링을 위해 인덱스가 필요합니다:

1. Firebase 콘솔 → Firestore Database → 인덱스
2. **인덱스 만들기** 클릭
3. 다음 설정:
   - 컬렉션 ID: `movies`
   - 필드 추가:
     - `genre` (오름차순)
     - `createdAt` (내림차순)
   - 쿼리 범위: 컬렉션
4. 생성 클릭 (몇 분 소요될 수 있음)

### 3. Storage 설정 (선택)

**Storage 없이 진행하는 경우:**
- 현재 상태로 계속 진행 가능
- 썸네일은 외부 URL로 입력
- 나중에 Storage 활성화 후 파일 업로드 기능 추가

**Storage 활성화하는 경우:**
- `STORAGE_SOLUTION.md` 파일 참고
- Blaze 플랜 업그레이드 필요
- Budget Alert 설정 필수

## 🧪 테스트 방법

1. **웹 앱 실행:**
   ```powershell
   cd web
   npm run dev
   ```

2. **테스트 시나리오:**
   - 회원가입 (제작자 역할)
   - 영화 업로드 (YouTube URL 입력)
   - 영화 목록에서 확인
   - 영화 상세 페이지 확인
   - 장르별 필터링 테스트

3. **주의사항:**
   - Firestore Rules를 설정하지 않으면 읽기/쓰기가 실패합니다
   - 인덱스를 생성하지 않으면 장르 필터링이 작동하지 않습니다
